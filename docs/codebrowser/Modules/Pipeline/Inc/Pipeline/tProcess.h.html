55 // Non-blocking. These constructors send messages to the parent including output and exit-code. This allows the 
56 // caller to continue working up till the point at which the destructor is called. At that point the destructor may 
57 // block if still necessary. The messages sent are listed above. In general as a user of this interface you'll do 
58 // some work while waiting for the ProcessDone message, at which point you may delete the object. User data gets 
59 // sent as the second param to ProcessDone, the exit code is the first. 
60 tProcess

69 );
