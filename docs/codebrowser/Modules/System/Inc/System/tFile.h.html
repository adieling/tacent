3// This file contains a class implementation of a file on a disk. It derives from tStream. By passing around tStreams 
4// any user code can be oblivious to the type of stream. It may be a file on disk, it may be a file in a custom 
5// filesystem, it may be a pipe, or even a network resource. 
6// 
7// Paths 
8// * A filesystem path may be represented by a tString or a tPath (not implemented yet but will essentially be a list of 
9// strings). 
10// * Paths when represented by strings use forward slashes as the separator unless it is the beginnging part of a 
11// Windows network shar in which case the first two seperators are \\ and \. 
12// eg. Posix/Linux path : "/home/username/work/important.txt 
13// eg. Windows file path : "C:/Work/Important.txt" 
14// eg. Windows net share : "\\machinename\sharename/Work/Important.txt" 
15// * A path can refer to either a file or a directory. If used for a directory it _always_ ends in a forward-slash /. 
16// * Input paths to functions here may use backslashes, but consistency in using forward slashes is advised. 
17// 
18// A note on variable naming of paths in this API. If it can be a file or directory, the word 'path' is used. If the 
19// path must be a directory, the word 'dir' is used. If the path must be a file, the word 'file' is used.
