124// The following print functions return the number of characters written to either stdout or to the supplied buffer. 
125// Format specification strings take the form below, with [] indicating an optional field. 
126// 
127// %[flags][width][.precision][:typeSizeElements][!typeSizeBytes][|typeSizeBits]type 
128// 
129// The format string is a superset of the ANSI standard for printf. It adds more types (vectors and matrices), 
130// consistent typesize fields and support for extra flags, including an alternative presentation flag. See 
131// http://www.cplusplus.com/reference/cstdio/printf/ for the standard types supported. 
132// 
133// [flags] 
134// Must be one or more of: - + SPACE 0 # _ ' 
135// 
136// Standard: hyphen (-) means left justify. Plus (+) means show sign for numeric types. # means force decimal 
137// point for e, E, f, g, and G. It also means prefix with 0, 0x, or 0X for o, x, and X. 
138// Using the 0 prefix flag works differently on Linux vs Windows with non-integral types like %c. 
139// Tacent behaves (on purpose) like Windows where the leading 0s are printed even though the type is 
140// not integral. 
141// Enhanced: Underscore (_) or single-quote (') chooses a more readable decorative or alternative printing 
142// method. They do not necessarily do the same thing as each other. 
143// Integral Types: 
144// For built-in integral types, an _ inserts an underscore every 4 characters starting from the 
145// right. Handy for making binary values readable. For Tacent integer types like tint128 and 
146// tuint256, the spacing is every 8 characters. 
147// For any integral type, a ' will place a comma every 3 digits starting from thr right. Makes 
148// big integers readable. eg. 23,456,789. This is not really an enhancement as of POSIX 2008. 
149// Matrix Types: 
150// A _ will print the 4x4 (or 2x2) matrix in 4 (or 2) separate rows surrounded by square [] 
151// brackets. Uses a default width of 9 and precision of 4, resulting in 1234.5678 for each 
152// element. Default expected matrix size is :16 (a 4x4 matrix). If you pass in a 2x2, you must 
153// specify the typesize. 
154// Vector Types: 
155// A _ causes just the components (space separated) to print. No decoration. By convention (a, b) 
156// is shorthand for the column vector: [ a 
157// b ] 
158// Default expected vector size is :3 (3 elements). If you pass in a 2 or 4 component vector you 
159// must specify the size. 
160// Bool Types: 
161// A _ results in the bool being printed as T or F. A single-quote ' results in Y or N.  
162// 
163// [width] 
164// A number representing the minimum num chars printed, or a * to indicate read width from the vararg list. 
165// 
166// [precision] 
167// A number representing decimal accuracy when printing floats, or a * to indicate read precision from the vararg list. 
168// 
169// [typeSize] 
170// A number after a colon (:) or a bang (!) or a pipe (|). This is not ANSI (ANSI ones are the odd h, l, ll, I32, I64). 
171// The colon means to interpret the number following as how many 32bit elements there are in the type. For example, :2, 
172// :3, :4 may be used with the vector 'v' type to indicate how many components are present. The bang (!) means 
173// interpret the number as a byte (octet I suppose, but really, what processor now days has a 7-bit byte? 32-bit chars 
174// are much more common). A value after a pipe symbol means the number is interpreted as a number of bits. For example, 
175// |32 or |64 may be used with integer types d, i, o, u, x, X, and b to specify the bit-size of the integer to print. 
176// 
177// [type] 
178// A single (case-sensitive) letter chosen from the types below: 
179// Integer: b o d i u x X For these types 'i' is same as 'd'. 'o' is octal. 'b' is binary. Handles Tacent 
180// integer types tint128, tint256, as well as all the built-in integral types. 
181// Pointer: p ISO C. Similar to %#010X (32bit systems) and %#018X (on 64bit systems) with the 
182// following two exceptions: 
183// a) The prefix 0x (lower case x) is used with capital ABCDEF (normally these 
184// are forced to match). 
185// b) The prefix is always used, even if the value is 0 (which is not the case 
186// for # in general). 
187// Float: f g Handles floats and doubles. 
188// Vector: v Use with :typesize for 2, 3, and 4 dimensional vectors. Default is 3. Must call 
189// pod(vec) or vec.Pod() for tVectors, or use tVecNb POD type. All other spec 
190// fields are per vector component. Not the whole thing. 
191// Quaternion: q Quaternions. Typesize is ignored. Must call pod for tQuaternions. Normal format is 
192// (w, x, y, z). Alternate _ format is (w, (x, y, z)). 
193// Matrix: m For use with tMatrix4 and tMatrix2 POD types. Matrix is printed on one line using 
194// () brackets. Use decorative _ flag for 4 (or 2) rows. All other spec fields are per 
195// vector component. The typesize is used to determine 2x2 or 4x4 (default). 
196// Character: c A single character. 
197// String: s You must call pod(string) or string.Pod() for tStrings, or use char*. 
198// t Windows only. Allows passing of non-POD tString directly. Warning: You cannot pass 
199// in an tStringItem! You must either cast to an tString or call pod() and use %s. 
200// Boolean: B For printing a bool as either "true" or "false" or the alternate formats T or F 
201// when _ used and Y or N when ' used. 
202// Percent: % Displays percent sign. 
203// 
204// The functions return the number of characters printed. They do NOT care about channels in that they always report 
205// what would have been printed if the channel was visible. Channels are an output filter only, not something that 
206// changes behaviour depending on what machine you are on. 
207// 
208// Examples: 
209// uint32 a = 0x1234ABCD; 
210// tPrintf("%08X", a); 
211// 
212// uint64 b = 0x1234ABCD5678ABCDLL; 
213// tPrintf("%016|64X or %016!8X or %016:2X", b, b, b); 
214// 
215// tuint256 c = 42; 
216// tPrintf("%064|256X or %064!32X or %064:8X", c, c);
