3// This clas knows how to load and save a JPeg (.jpg and .jpeg) file. It does zero processing of image data. It knows 
4// the details of the jpg file format and loads the data into a tPixel array. These tPixels may be 'stolen' by the 
5// tPicture's constructor if a jpg file is specified. After the array is stolen the tImageJPG is invalid. This is 
6// purely for performance.

117 // A perfect lossless transfrom is one where the area of the image is the same before and after the transform. 
118 // An imperfect lossless transform is still lossless, but some edges of the image need to be culled. For all 
119 // lossless transforms (flips/rotates) to be perfect two things must be true: 
120 // 
121 // a) The NoDecompress load-flag must have been used. 
122 // b) The image's width and height must be evenly divisible by the MCU block size. 
123 // 
124 // For b) if both width and height are divisible, all transforms are possible. If one is divisible then the 
125 // transform may be possible or it may not be (depending on the transform). This is why the specific transform must 
126 // be supplied. If false you can still perform a LosslessTransform, but one or two outer edges will be culled. 
127 bool CanDoPerfectLosslessTransform(Transform) const; 
128 
129 // If allowImperfect is true you may end up with a slightly cropped image. This cropping will happen if 
130 // CanDoPerfectLosslessTransform returned false. If allowImperfect is false, this function will return false and do 
131 // nothing unless it can guarantee no cropping. 
132 bool LosslessTransform(Transform, bool allowImperfect = true);
