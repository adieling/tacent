3// This module contains script file readers and writers. Tacent supports two text script formats. The main one is in 
4// the spirit of Church's lambda calculus and used 'symbolic expressions'. ex. [a b c]. See tExpression. I personally 
5// find this notation much more elegant than stuff like the lame XML notation. This site agrees: 
6// http://c2.com/cgi/wiki?XmlIsaPoorCopyOfEssExpressions 
7// The tExpression reader class in this file parses 'in-place'. That is, the entire file is just read into memory once 
8// and accessed as const data. This reduces memory fragmentation but may have made implementation more complex.  
9// 
10// The second format is a functional format. ex. a(b,c) See tFunExtression.

39// An s-expression has the following syntax: [expr expr ...] OR atom. That is, an s-expression is either a list of 
40// s-expressions enclosed in square brackets or it is an 'atom' (base value type, not divisible, as in an atom). 
41// By convention s-expressions often take the more restricted form [command argument1 argument2 argument3] where the 
42// the command is a string atom and arguments are arbitrary expressions. At the leaf level an expression is an atom. 
43// 
44// In comparison to Lisp expressions made of pairs, we support the list-syntax, not the pair syntax. eg. Lisp's 
45// (a.(b.(c.()))) where () is null is represented as [a b c]. Cdr is not much use in proper lists, but Car, Cadr, 
46// Caddr, etc are. i.e. Car([a b c]) = a. Cadr([a b c]) = b. etc.

91 // These get the values of atom expressions. They are the base types. eg: the atom 'True' would return true if you 
92 // called GetAtomBool on it. '67' would return 67 if you called GetAtomInt, etc. It is worth mentioning GetAtomFloat 
93 // and GetAtomDouble. They can take atoms of the form '67.3677' or '67.3677#FB0933CE'. The value after the # is a 
94 // base-16 number that is the binary representation of a float (IEEE 754). It would be twice as long for a double. 
95 // If there is a # and hex value, it is used instead of the base 10 text. This essentially removes 'wobble' like 
96 // you see in many engines from saving and reloading floats, and keeps it human-readable/editable. Note, I have 
97 // very little confidence FB0933CE actually represents 67.3677 -- I just chose a random 32 bit hex number.

203// Use this to read and parse an existing script. A script file is a list of expressions without []'s around the 
204// entire file. e.g. This is a valid script: 
205// 
206// [a b c] ; Arg0 
207// d ; Arg1 
208// [e f] ; Arg2 
209// 
210// You can pass one of these to any function that expects a tExpression as it is a sub-class. 
211class tExprReader : public tExpression 
212{ 
213public: 
214 // Constructs an initially invalid tExprReader. 
215 tExprReader() : tExpression(), ExprBuffer(nullptr) { } 
216 
217 // If isFile is true then the file 'name' is loaded, otherwise treats 'name' as the actual script string. 
218 tExprReader(const tString& name, bool isFile = true) : tExpression(), ExprBuffer(nullptr) { Load(name, isFile); } 
219 
220 // Useful for command line utilities. Makes a script from standard command line argc and argv parameters. Honestly, 
221 // I'm not sure how useful this is now that we have tOption for parsing command lines in a nice way that is a bit 
222 // more standard. 
223 tExprReader(int argc, char** argv); 
224 ~tExprReader() { delete[] ExprBuffer; } 
225 
226 // If isFile is true then the file 'name' is loaded, otherwise treats 'name' as the actual script string. The 
227 // object is cleared before the new information is loaded. Any previous information is lost. 
228 void Load(const tString& name, bool isFile = true); 
229 
230 // The object will be invalid after this call. 
231 void Clear() { delete[] ExprBuffer; ExprBuffer = 0; } 
232 bool IsValid() const { return ExprBuffer ? true : false; }

234private: 
235 // ExprBuffer is officially a UTF-8 string. i.e. We support unicode codepoints in script files.

340 // The Comp (Compose) functions write a list of 2 to 5 atoms (a command followed by 1 to 4 arguments). They are 
341 // convenience functions to write expressions of the form [s a b] followed by a carraige return. The first atom is
